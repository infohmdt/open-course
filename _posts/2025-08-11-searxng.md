## Pendahuluan  
SearXNG adalah [metasearch engine](https://en.wikipedia.org/wiki/Metasearch_engine) sumber terbuka yang mengagregasi hasil dari berbagai penyedia pencarian (termasuk Google, Bing, dan DuckDuckGo) tanpa menyimpan data pengguna atau melacak aktivitas. Solusi [self-hosted](https://en.wikipedia.org/wiki/Self-hosting_(web_services)) memungkinkan pengguna mengontrol infrastruktur pencarian secara penuh, dengan [Docker](https://ricaldocs.github.io/tags/docker/) menyederhanakan proses penyebaran melalui kontainerisasi. Implementasi ini menjamin privasi pencarian, menghindari sensor eksternal, dan memfasilitasi pemeliharaan sistem.

## Petunjuk Penggunaan  
Terdapat dua metode untuk menghosting SearXNG:
 
1. Tidak memerlukan pengetahuan awal tentang [self-hosting](https://en.wikipedia.org/wiki/Self-hosting_(web_services)) dan direkomendasikan untuk pemula. Metode ini menyertakan Caddy sebagai reverse proxy dan otomatis menangani sertifikat TLS. 
2. Ditujukan untuk pengguna berpengalaman yang telah memiliki reverse proxy sendiri (misalnya Nginx, HAProxy) dan layanan lain di mesin mereka.

### Instalasi
1. [Instal Docker](https://docs.docker.com/install/)  
2. Unduh image terbaru:
   ```bash
   docker pull docker.io/searxng/searxng:latest
   ```

3. Unduh searxng-docker:  
   ```bash
   cd /usr/local
   ```

   ```bash
   git clone https://github.com/searxng/searxng-docker.git && cd searxng-docker
   ``` 

4. Edit berkas `.env`{: .filepath} untuk mengatur hostname dan email 

5. Generate secret key:  
   ```bash
   sed -i "s|ultrasecretkey|$(openssl rand -hex 32)|g" searxng/settings.yml 
   ```

### Metode 1: Dengan Caddy (disarankan untuk pemula)  
6. Jalankan SearXNG di background:  
   ```bash
   docker compose up -d
   ```  

   > Sekarang SearXNG dapat diakses di `localhost:8080`
   {: .prompt-info}

### Metode 2: Reverse Proxy Pribadi (pengguna berpengalaman)  
1. Hapus bagian terkait Caddy di `docker-compose.yaml`{: .filepath} (layanan dan volume-nya)  
2. Arahkan reverse proxy Anda ke port layanan `searxng` di `docker-compose.yml`{: .filepath} (bawaan: 8080)  
3. Hasilkan dan konfigurasikan sertifikat TLS dengan reverse proxy pilihan Anda  
4. Jalankan SearXNG di background:  
   ```bash
   docker compose up -d
   ```  

   > Port internal `searxng` dapat diubah melalui variabel lingkungan `BIND_ADDRESS` (misal untuk mode jaringan `host`). Atur variabel ini langsung di `docker-compose.yaml`{: .filepath}. 
   {: .prompt-tip}

## Volumes
Dua volume diekspos yang harus dipasang untuk menjaga isinya:
- `/etc/searxng`{: .filepath}: Berkas konfigurasi (`settings.yml`{: .filepath}, etc.)
-  `/var/cache/searxng`{: .filepath}: Persistent data (`faviconcache.db`{: .filepath}, etc.)

## Environment Variables
Variabel lingkungan berikut dapat dikonfigurasi:
- `$SEARXNG_*`: Mengontrol opsi konfigurasi SearXNG, perhatikan lingkungan `$SEARXNG_*` di [server](https://docs.searxng.org/admin/settings/settings_server.html#settings-server) dan [general](https://docs.searxng.org/admin/settings/settings_general.html#settings-general).
- `$GRANIAN_*`: Mengontrol [opsi server Granian](https://docs.searxng.org/admin/installation-granian.html#granian-configuration).
- `$FORCE_OWNERSHIP`: Memastikan volume/berkas yang dipasang dimiliki oleh pengguna `searxng:searxng` (default: `true`)

Jalur internal kontainer (jangan diubah kecuali Anda tahu apa yang Anda lakukan):
- `$CONFIG_PATH`: Jalan ke direktori konfigurasi SearXNG (default: `/etc/searxng`{: .filepath})
- `$SEARXNG_SETTINGS_PATH`: Jalan ke berkas pengaturan SearXNG (default: `$CONFIG_PATH/settings.yml`)
- `$DATA_PATH`: Jalan ke direktori data SearXNG (default: `/var/cache/searxng`{: .filepath})

## Troubleshooting - Akses Log  
- Akses log semua kontainer:  
```shell
docker compose logs -f
```  
- Log spesifik:  
  - Caddy: `docker compose logs -f caddy`  
  - SearXNG: `docker compose logs -f searxng`  
  - Valkey: `docker compose logs -f redis`  

### Menjalankan SearXNG dengan systemd  
Langkah ini opsional untuk pengguna non-systemd.  
1. Salin template layanan:  
   ```bash
   cp searxng-docker.service.template searxng-docker.service
   ```  

2. Sesuaikan `WorkingDirectory` di `searxng-docker.service` (jika jalur instalasi bukan `/usr/local/searxng-docker`{: .filepath})  

3. Aktifkan layanan:  
   ```bash
   systemctl enable $(pwd)/searxng-docker.service
   ```  

4. Mulai layanan:  
   ```bash
   systemctl start searxng-docker.service
   ``` 

   > **Penting**: Verifikasi jalur berkas layanan sebelum mengaktifkannya.
   {: .prompt-tip}
